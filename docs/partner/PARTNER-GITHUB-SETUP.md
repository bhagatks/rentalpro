# GitHub setup — invite your partner

**See also:** [PARTNER-START-HERE.md](./PARTNER-START-HERE.md)

Both of you are partners. GitHub just needs one person to send the invite — whoever created the repository can do that in Settings.

---

## If you need access

1. Create a GitHub account with your **personal Gmail**
2. Send your **GitHub username** to your **other partner**
3. Check Gmail (and spam) for the GitHub invite → **Accept**
4. Clone the repo with GitHub Desktop → start at `PARTNER-START-HERE.md`

Message template:

> GitHub username: `[your-username]`  
> Email: `[your-gmail@gmail.com]`

---

## If you created the repo — invite your partner

1. Open the **rentalpro** repository on GitHub
2. **Settings** → **Collaborators**
3. **Add people** → enter your partner’s GitHub username
4. Role: **Write** → confirm
5. Tell your partner to check email and accept

**Done when:** Your partner confirms they can open the repo.

---

## After the invite works

| Step | Both partners |
|------|---------------|
| Clone | GitHub Desktop → clone **rentalpro** |
| Work | Branch → edit docs → commit → push |
| Handoff | Open a **Pull Request** → @mention your partner for review |
| Sync | After merge → **Fetch → Pull** in GitHub Desktop |

Use **GitHub Issues** for decisions and questions — see Part 2B in [PARTNER-START-HERE.md](./PARTNER-START-HERE.md).
